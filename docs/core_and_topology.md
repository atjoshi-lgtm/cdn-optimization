# Core Configuration and Topology

This section covers how the application is configured and how it understands the physical network map.

## Core Configuration (`src.cdn_optimizer.core.config_loader`)
The application relies on `config/default_config.yaml` instead of hardcoded variables. 
- The `load_config()` function parses the YAML and returns a strongly-typed `CDNConfig` dataclass.
- File paths are automatically converted to `pathlib.Path` objects.

## Data Access: CSV Parser (`src.cdn_optimizer.data_access.csv_parser`)
- **`load_metro_ids`**: Reads `metro_areas.csv`. It specifically extracts the integer ID from Column 0 and the string name from Column 1. 

## Topology: Network Map (`src.cdn_optimizer.topology.network_map`)
- **`MetroResolver`**: Acts as the boundary between string-based script inputs and integer-based database queries. It accepts the dictionary generated by the CSV parser and strictly enforces lookups, throwing a `KeyError` if an invalid metro is requested.
